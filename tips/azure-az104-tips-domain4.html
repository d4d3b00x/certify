Patrones que puntúan

  • Segmentación clara: subredes por función, NSG por capa, UDR para control de rutas.
  • Conectividad: VNet peering (intra/region) y VPN/ExpressRoute para on‑prem; Azure Firewall o NVA para inspección.
  • Balanceo: ALB/AGW según capa 4/7; Health Probes correctos.
  • Privacidad: Private Endpoints a PaaS; DNS propio con Azure DNS/Private Resolver.

Errores comunes

  • No considerar límites de emparejamiento o transitive routing.
  • Endpoints públicos abiertos a toda Internet.

Lista de estudio

  • VNet Peering vs VPN Gateway, y escenarios de Hub‑&‑Spoke.
  • Network Watcher, Connection Monitor y captura de paquetes.

Pista de examen

Si necesitas PaaS accesible de forma privada, prioriza Private Endpoint sobre IP pública.
