Definir objetivos: RPO y RTO

  • RPO: pérdida máxima de datos aceptable.
  • RTO: tiempo máximo de indisponibilidad.
  • Clasifica aplicaciones (Tier 0/1/2) y alinea coste vs objetivo.

Catálogo de opciones HA/DR

Componente Opción RPO/RTO típico Notas
Máquinas virtuales ASR a región secundaria minutos / horas Runbooks, orden de arranque y pruebas no disruptivas.
SQL PaaS Auto-failover groups ~0 / ~30–60s Cadena de conexión y reintentos.
Cosmos DB Multi-región (read/write) segundos / <1min Consistencia adecuada + resolución de conflictos.
Storage GZRS/RA-GZRS + versioning minutos / n/a Protección ante pérdida de zona y borrados accidentales.

Azure Site Recovery (ASR)

  • Replica VMs (Azure, VMware, Hyper-V) a otra región; snapshots consistentes con app.
  • Define Recovery Plans (grupos, secuencia, scripts, manual steps).
  • Pruebas non-disruptive en red aislada; reportes de cumplimiento.
  • Integra con Private Endpoints, NSG, balanceadores, IP estática vs dinámica.

Backup, retención e inmutabilidad

  • Azure Backup para VMs, Files, SAP HANA, SQL en VM; vault con soft-delete e inmutabilidad.
  • SQL PaaS: puntos de restauración automáticos + LTR (hasta 10 años).
  • Blob: versioning, change feed, immutability policy (legal hold/time-based).
  • Prueba de restauración periódica (runbooks) y documentación de tiempos reales.

Zonas de disponibilidad y regiones emparejadas

  • Zonas: separa fallos dentro de la región (LB Standard, Zonal VMs, Zone-redundant PaaS).
  • Paired region: actualizaciones coordinadas, prioridad en recuperación.
  • Datos soberanía/regulatorio: asegúralo con Policy + ubicación del vault/lake.

Runbooks, monitorización y pruebas

  • Runbooks declarativos (Automation/Functions) para conmutación y reversión.
  • Alertas: métricas de réplica/cola, fallas, latencias; Tablero de DR.
  • Pruebas de chaos controladas y simulacros anuales como mínimo.

Preguntas tipo examen

  1. “Recuperar VMs ante caída regional” → ASR + Recovery Plans + pruebas no disruptivas.
  2. “RPO≈0 y RTO<1min para SQL PaaS” → Auto-failover groups + reintentos.
  3. “Proteger ficheros ante ransomware” → Azure Backup con inmutabilidad y soft-delete.
  4. “Logs críticos siempre disponibles” → GZRS + immutability / versioning.
