Patrones que puntúan

  • Tipo de cuenta correcto: StorageV2 para funciones modernas; Premium para latencia baja.
  • Redundancia: LRS para coste, ZRS para alta disponibilidad zonal, GRS/RA‑GRS para DR regional.
  • Seguridad: puntos de conexión privados, Azure AD auth para Azure Files, cifrado con claves gestionadas por cliente (CMK) cuando sea necesario.
  • Ciclo de vida: reglas para mover de Hot→Cool→Archive y eliminar versiones antiguas.

Errores comunes

  • Exponer Blob públicamente sin restricciones.
  • Usar SAS de larga duración sin stored access policy.

Lista de estudio

  • Comparar Azure Files vs Blob; instantáneas, replicación y rendimiento.
  • AzCopy/Azure Storage Explorer y características de NFS/SMB.

Pista de examen

Para compartir archivos con identidades de AD: Azure Files + Azure AD DS o Kerberos.
