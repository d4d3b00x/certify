Patrones que puntúan

  • Dimensiona con métricas: usa autoscaling por CPU/QPS/latencia; reserva o Savings Plans para cargas estables.
  • CDN y caching: CloudFront + S3/ALB, TTL adecuados, Lambda@Edge para encabezados/cookies.
  • Almacenamiento correcto: EBS gp3/io1 para IOPS, S3 para objetos, EFS para compartidos (Burst/Turbo), FSx para performance de Windows o Lustre.
  • Base de datos adecuada: DynamoDB para baja latencia y escalado; RDS/Aurora para SQL (read replicas, Global Database).
  • Procesamiento paralelo: SQS + Lambda/Batch/Fargate; particiona y usa fan‑out con SNS.

Errores comunes

  • EBS gp2 con cuello de botella de burst.
  • Una sola réplica de lectura para tráfico global.
  • No usar compresión/HTTP2/keep‑alive en ALB/CloudFront.

Lista de estudio

  • Diferencias entre gp2/gp3/io1/io2; throughput vs IOPS.
  • CloudFront behaviors, orígenes múltiples, funciones en el borde.
  • Escalado de DynamoDB (on‑demand/provisioned, autoscaling, DAX, GSIs/LSIs).

Pista de examen

Para lecturas globales de baja latencia: CloudFront o Aurora Global Database. Para claves‑valor con milisegundos, DynamoDB + DAX.
