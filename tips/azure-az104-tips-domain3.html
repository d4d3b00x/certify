Azure AZ-104 — Domain 3 (Deploy & Manage Compute Resources)

Scale out smart, secure, and cost-aware compute

Domain 3 (about 25-30% of the exam) covers deploying and configuring VMs, VMSS, containers, and Azure App Service. You’ll also handle availability sets/zones, extensions, automation, and custom images.

Patterns that score

  • Use VMSS > single VM for scale and self-healing. Combine with Load Balancer or App Gateway for traffic distribution.
  • Deploy via templates (ARM/Bicep) to ensure consistency; use parameters for region, SKU, tags, etc.
  • Choose the right availability: availability set (domain+update domains) vs availability zones for true AZ redundancy.
  • Use managed identities instead of credentials to access resources (Key Vault, Storage, etc.).
  • Automate config with Custom Script Extension, DSC, or Azure Automation State Configuration.
  • Deallocate ≠ delete: Stopped (deallocated) releases compute cost but keeps storage billing.
Exam hint: Pick the solution that reduces manual management and improves availability or compliance.

Common pitfalls

  • Creating unmanaged disks or forgetting to set OS disk type (Premium SSD vs Standard SSD/HDD).
  • Forgetting NSG association on both subnet and NIC levels.
  • Not enabling boot diagnostics or log analytics agent before troubleshooting questions.
  • Mixing up App Service Plan tiers (Shared/Basic vs Standard/Premium) for scaling & SSL support.
  • Confusing managed identity types: system-assigned vs user-assigned.

Study checklist

  • VM creation options (Portal, CLI, PowerShell, templates).
  • Availability Sets vs Availability Zones vs VM Scale Sets.
  • Managed Disks (Standard/Premium/Ultra) and snapshots/images.
  • Extensions: Custom Script, DSC, Azure Monitor Agent, Dependency Agent.
  • Automation: Runbooks, Update Management, Desired State Configuration.
  • Containers: Azure Container Instances, Web App for Containers, AKS basics.
  • App Service: scaling, deployment slots, configuration, identity, backup.

CLI quick hits

  • Create VM:
    az vm create -g rg-core -n vm1 --image UbuntuLTS --admin-username azureuser --generate-ssh-keys --zone 1
  • Deallocate:
    az vm deallocate -n vm1 -g rg-core
  • Attach data disk:
    az vm disk attach -g rg-core --vm-name vm1 --name datadisk1 --size-gb 64
  • Create Scale Set:
    az vmss create -n vmss-web -g rg-core --image UbuntuLTS --instance-count 2 --upgrade-policy automatic
  • Deploy ARM template:
    az deployment group create -g rg-core --template-file main.json --parameters @params.json
  • Create App Service Plan + Web App:
    az appservice plan create -g rg-core -n asp-prod --sku P1v2 --is-linux
    az webapp create -g rg-core -p asp-prod -n mywebapp --runtime "NODE:18LTS"
  • Create container instance:
    az container create -g rg-core -n helloaci --image mcr.microsoft.com/azuredocs/aci-helloworld --cpu 1 --memory 1 --ports 80

Exam Focus

  • Deploy VMs and configure availability, extensions, disks, and identities.
  • Configure and manage VMSS scaling and upgrades.
  • Deploy and manage containers (ACI, AKS basics).
  • Deploy and configure App Service (web apps, plans, scaling, backups).
  • Automate VM deployment and configuration (ARM/Bicep, Automation, Runbooks).

Key Azure services

  • Virtual Machines, Managed Disks, Snapshots, Images
  • VM Scale Sets, Availability Sets/Zones
  • Azure App Service + Plans, Web App for Containers
  • Azure Container Instances (ACI), AKS basics
  • Azure Automation, Update Mgmt, Log Analytics

Hands-on ideas

  • Deploy a VMSS with a Custom Script Extension to install NGINX.
  • Create a snapshot → image → new VM from image.
  • Enable boot diagnostics and connect serial console for troubleshooting.
  • Deploy App Service with staging slot + swap operation test.
  • Configure user-assigned managed identity and grant it Key Vault access.
