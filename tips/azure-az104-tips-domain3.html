Máquinas virtuales, scale sets, imágenes y disponibilidad.

Patrones que puntúan

  • Disponibilidad: Zonas de disponibilidad o Conjuntos de disponibilidad; balanceo con Azure Load Balancer/ALB.
  • Escalado: VMSS con reglas de CPU, cola o métrica personalizada; custom image o Shared Image Gallery.
  • Gestión: extensiones de VM, Update Manager y Azure Monitor para inventario y parches.

Errores comunes

  • Una sola instancia crítica sin SLA.
  • No separar datos de SO en discos gestionados distintos (OS vs datos).

Lista de estudio

  • Tipos de disco (Standard/Premium SSD, Ultra), IOPS/throughput y emparejamiento.
  • Backup de VM y restauraciones a nivel de archivo/disco.

Pista de examen

Para consistencia a escala, piensa en VMSS + imágenes compartidas.
