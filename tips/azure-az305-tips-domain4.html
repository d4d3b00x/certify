Compute: cuándo elegir qué

App Service / Functions

  • HTTP/API, CI/CD rápido, autoscale; Deployment Slots para azul/verde.
  • Isolated v3 para redes privadas; Private Endpoints al backend.

AKS (Kubernetes)

  • Microservicios/containers; node pools por tipo; autoscaler; Spot para trabajos batch.
  • Ingress + AGIC / NGINX; identidad gestionada; CSI Secrets con Key Vault.

VMs / VM Scale Sets

  • SO/soft específico, stateful; VMSS para escala horizontal.
  • Galería de imágenes compartidas; Extensiones (Custom Script/DSC).

Coste

  • Reserved Instances/Savings Plans para cargas estables.
  • Spot para no críticos; autoapagado, auto-shrink.

Balanceo global/regional y WAF

Servicio Capa Uso típico
Azure Front Door Global L7 + WAF + CDN Web global, failover geográfico, aceleración.
Application Gateway (WAF) Regional L7 Routing HTTP/HTTPS, mTLS, end-to-end TLS.
Azure Load Balancer L4 (TCP/UDP) zonal/global Backends VMs/VMSS; puertos no HTTP; HA interna/externa.

VNet, Private Link y DNS

  • Arquitectura: Hub-Spoke, peering con Use Remote Gateway según topología.
  • Private Endpoints a PaaS; deshabilita acceso público; usa Private DNS Zones.
  • Nombres: DNS privado enlazado a spoke; conditional forwarders on-prem.
  • NSG/ASG por segmentos; Azure Firewall para egress control y FQDN tags.

Conectividad híbrida

  • VPN para arranque/DR; ExpressRoute para producción estable.
  • BGP recomendado; Gateway SKUs correctas (ErGw3AZ para alto rendimiento).
  • Rutas: propagación controlada, UDR para egress a Firewall.

Seguridad y cumplimiento

  • Zero-trust: identidad (Managed Identity), least privilege, Policy para baseline.
  • Key Vault con Private Endpoint; rotación automática de secretos/certificados.
  • Defender for Cloud recomendaciones + Just-in-Time VM access.

Disponibilidad y escala

  • Zonal: distribuye VM/VMSS por zonas; LB Standard zonal o zone-redundant.
  • Autoscale: métricas de CPU/cola; reglas por horario y picos.
  • Estado: para stateful usa discos compartidos/managed disks + Availability Sets.

Observabilidad

  • Diagnostic Settings a Log Analytics; métricas y logs con retención definida.
  • Alertas en Action Groups; Workbooks por dominio (red, compute, seguridad).
  • Service Health + Resource Health para RCA y SLA.

Preguntas tipo examen

  1. “Web global con WAF y failover” → Front Door (WAF) + backends en varias regiones.
  2. “Balancear TCP interno entre VMs” → Load Balancer Standard interno.
  3. “Privatizar PaaS” → Private Endpoints + Private DNS Zones.
  4. “Minimizar coste sin perder capacidad” → Autoscale + Reserved/Savings + Spot para batch.
