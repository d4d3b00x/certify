1. RBAC por ámbito

Asigna a **grupos** y no a usuarios. Evita **Owner**; usa **User Access Administrator** para delegar.

ÁmbitoUso típico
ÁmbitoEjemplo
Management GroupPolíticas globales
SuscripciónRoles de facturación
RG/RecursoAcceso a un servicio puntual

2. Privileged Identity Management

Habilita acceso **JIT** con aprobación; registra auditoría y define duración mínima.

EXAM TIP: “Permisos temporales” → PIM + aprobación.

3. Managed Identities

Identidad administrada asignada a recursos para solicitar tokens AAD sin secretos. Ideal para acceder a **Key Vault** o APIs.

4. Key Vault

Activa **soft‑delete** y **purge protection**; separa bóvedas por entorno y aplica RBAC a nivel de datos.
