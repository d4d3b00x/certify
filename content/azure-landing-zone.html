Arquitectura de Landing Zone en Azure

Referencia de diseño para organizar identidades, políticas, red y suscripciones en una zona de aterrizaje empresarial.

Resumen

Una Landing Zone estandariza cómo desplegar cargas en Azure: jerarquía de Management Groups, separación por suscripciones, red hub-spoke, identidad y políticas que aplican seguridad por defecto.

AZUREBlueprint de referencia

Índice

1. Management Groups y suscripciones 2. Identidad y RBAC 3. Azure Policy y gobernanza 4. Red: Hub-Spoke + conectividad 5. Operación: Monitor, Backup, Costes 6. Seguridad de base 7. Despliegue rápido (Bicep) 8. Checklist

1) Management Groups y suscripciones

  • Raíz ➜ contoso; hijos: platform, corp, online.
  • Suscripciones: separar connectivity, identity, management (plataforma) y una por dominio de negocio.
  • Políticas y budgets heredados en MG; excepciones a nivel de suscripción.

2) Identidad y RBAC

  • Azure AD como autoridad. Grupos de seguridad mapeados a roles: Owner (plataforma limitada), Contributor (apps), Reader.
  • Usa Privileged Identity Management y acceso just-in-time.
  • Identidades administradas para workloads; evita secretos en código.

3) Azure Policy y gobernanza

  • Asignar policy sets: etiquetado obligatorio, regiones permitidas, sku aprobadas, Defender habilitado.
  • Denegar IP públicas en subredes sensibles, exigir Azure Backup y cifrado.

4) Red: Hub-Spoke + conectividad

  • Hub: Firewall/NSG, Azure Bastion, VPN/ExpressRoute, DNS privado.
  • Spokes por dominio: subredes web, app, data con NSG.
  • Emparejamiento peering controlado y UDR para inspección por Firewall.
# Ejemplo UDR (az cli)
az network route-table route create   --route-table-name rt-spoke-app --resource-group rg-net   --name default-to-fw --address-prefix 0.0.0.0/0 --next-hop-type VirtualAppliance --next-hop-ip-address 10.0.0.4

5) Operación: Monitor, Backup, Costes

  • Log Analytics + Azure Monitor (alerteras y dashboards).
  • Azure Backup y políticas de retención por tipo de dato.
  • Cost Management con budgets y tags costCenter.

6) Seguridad de base

  • Microsoft Defender for Cloud con recomendaciones secure score.
  • Just-in-Time para VM, bloqueo de Public Network Access en PaaS.
  • Clave gestionada por cliente (CMK) para datos sensibles.

7) Despliegue rápido (Bicep)

// Hub virtual network (bicep)
param location string = resourceGroup().location
resource vnet 'Microsoft.Network/virtualNetworks@2023-09-01' = {
  name: 'vnet-hub'
  location: location
  properties:{
    addressSpace:{ addressPrefixes: ['10.0.0.0/16'] }
    subnets:[
      { name:'AzureFirewallSubnet', properties:{ addressPrefix:'10.0.0.0/24' } },
      { name:'GatewaySubnet', properties:{ addressPrefix:'10.0.1.0/27' } },
      { name:'shared', properties:{ addressPrefix:'10.0.10.0/24' } }
    ]
  }
}

8) Checklist

  • ¿Tienes MG y suscripciones separadas por plataforma/negocio?
  • ¿Policies aplicadas en MG con excepciones mínimas?
  • ¿Hub-Spoke con inspección y DNS privado centralizados?
  • ¿PIM, Backup, Defender y budgets habilitados?
