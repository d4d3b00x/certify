Seguridad y Zero Trust en Azure (AZ-305)

ÍNDICE

1. Principios Zero Trust

Zero Trust asume que nada dentro o fuera del perímetro es confiable. Toda conexión se valida explícitamente.

  • Verificar explícitamente: autenticación fuerte y continua.
  • Usar acceso con privilegio mínimo: roles temporales y segmentación.
  • Asumir brecha: telemetría, detección y respuesta continua.
Zero Trust ≠ una herramienta, es una estrategia arquitectónica.

2. Identidad y control de acceso

  • Microsoft Entra ID: identidad central con autenticación multifactor (MFA) y Conditional Access.
  • Privileged Identity Management (PIM): acceso just-in-time para roles administrativos.
  • Managed Identities: evitan credenciales en código.
  • Access Reviews y Entitlement Management para gobernanza de usuarios.
Ejemplo: PIM activa Owner 1h → MFA → registro de auditoría

3. Protección de red y endpoints

  • Azure Firewall Premium: inspección TLS, filtrado FQDN y categorías URL.
  • Network Security Groups (NSG): reglas L4 por subred o NIC.
  • Azure DDoS Protection: capa perimetral.
  • WAF: protege apps web en App Gateway o Front Door.
  • Just-In-Time Access: Defender habilita acceso temporal a VMs.

4. Cifrado y protección de datos

  • Azure Key Vault: almacena claves, secretos y certificados.
  • Encryption at rest: Storage y SQL usan AES-256.
  • Encryption in transit: TLS 1.2/1.3 obligatorio.
  • Confidential Compute: protege datos en uso con enclaves SGX.

5. Monitoreo y respuesta

  • Microsoft Defender for Cloud: postura, recomendaciones y alertas.
  • Sentinel: SIEM/SOAR nativo con correlación de eventos.
  • Secure Score: medición global del cumplimiento de seguridad.

6. Checklist de examen

  • Zero Trust = verificar, minimizar, asumir brecha.
  • MFA + PIM + Conditional Access = obligatorio.
  • Firewall Premium + NSG + DDoS + WAF.
  • Key Vault + cifrado AES-256 + Confidential Compute.
  • Defender + Sentinel + Secure Score.
