1. Organizations & OU

Separa cuentas por entorno y línea de producto; habilita facturación consolidada, delegación de administración y aislamiento de blast radius. Para un aterrizaje guiado, usa AWS Control Tower para crear OUs, cuentas y guardrails iniciales.

ObjetivoAcción
Aislar riesgosCuentas separadas por entorno (prod, pre, dev) y por dominio de negocio.
Estándar inicialDespliegue con Control Tower (landing zone, OUs, cuentas y guardrails).
Gobierno centralOU jerárquicas, delegación y políticas a nivel organización.

2. SCP como guardrails

Las SCP (Service Control Policies) no conceden permisos; definen el máximo permiso efectivo que una principal puede tener dentro de una cuenta. Úsalas para bloquear regiones no permitidas, exigir encriptación o evitar configuraciones públicas.

EXAM TIP: “Evitar que creen recursos públicos” → SCP de denegación + S3 Block Public Access + políticas adecuadas.

3. Buenas prácticas IAM

  • Least privilege con políticas administradas por uso y revisión periódica.
  • Roles entre cuentas para workloads; evita credenciales largas. Usa STS (credenciales temporales).
  • MFA obligatoria para humanos; password policy y rotación de claves para integraciones heredadas.
  • Para identidades humanas, considera IAM Identity Center (SSO) con grupos y asignaciones a cuentas/roles.

4. Auditoría

Habilita CloudTrail a nivel organización con envío a una cuenta de seguridad. Complementa con AWS Config (reglas y drifts) y Security Hub para postura continua.
