2. SCP como guardrails

Las **SCP** limitan lo que se puede hacer en una cuenta (no conceden permisos). Úsalas para bloquear recursos públicos o regiones no permitidas.

EXAM TIP: “Evitar que creen recursos públicos” → SCP + S3 Block Public Access.

3. Buenas prácticas IAM

Principio de **least privilege**, **roles** entre cuentas, **MFA** para humanos y **STS** para credenciales temporales.

4. Auditoría

**CloudTrail** a nivel organización, **AWS Config** y **Security Hub** para postura continua.
