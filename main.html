AWS Certified Solutions Architect — Associate (SAA-C03)

Certify Crash Guide

One-page recap of core services, patterns, and exam-day heuristics.

Exam Domains

  • Domain 1 — Design Secure Architectures (30%): IAM, KMS, Secrets Manager, S3 policies, VPC endpoints, WAF/Shield.
  • Domain 2 — Design Resilient Architectures (26%): Multi-AZ/Region, ALB + Auto Scaling, RDS/Aurora features, Route 53, CRR.
  • Domain 3 — Design High-Performing Architectures (24%): Caches (ElastiCache/DAX), EBS gp3/io2, Global Accelerator, Athena + Parquet.
  • Domain 4 — Design Cost-Optimized Architectures (20%): Right-sizing, Savings Plans, Intelligent-Tiering, VPC endpoints (evitar NAT).

Aprende defaults de cada servicio, límites, y dónde están los límites de seguridad (cuenta/VPC/SG/bucket/key).

Quick Patterns

  • S3 privado: Gateway Endpoint + bucket policy aws:SourceVpce.
  • EKS least-privilege: IRSA (OIDC) con rol por service account.
  • Serverless privado: Interface Endpoints (Secrets Manager, SSM, STS…).
  • Baja latencia global: Global Accelerator para tráfico no-cacheable.
  • Base de datos HA: RDS Multi-AZ; replicas para lectura; Aurora = failover rápido.

Compute & Containers

  • EC2: familia por necesidad; Auto Scaling; SG + IAM roles; On-Demand/Spot/RIs/Savings Plans.
  • Lambda: paga por request/100ms; triggers (S3, API GW, SQS, DynamoDB); DLQs y retries.
  • ECS/EKS: ECS sencillo; EKS = Kubernetes; Fargate serverless; imágenes en ECR.

Prefiere servicios gestionados salvo que pidan acceso OS o networking muy custom.

Networking

  • Public vs Private subnets (IGW vs NAT). SGs = stateful; NACLs = stateless.
  • Acceso privado a AWS: Gateway endpoints (S3/DynamoDB), Interface endpoints (resto).
  • Entre VPCs: PrivateLink (NLB productor), VPC Peering (no transitivo), TGW hub-and-spoke.

Storage & Data

  • S3: versioning, encryption (SSE-S3/KMS), lifecycle, replication, access points; clases: Standard / IA / One Zone-IA / Intelligent-Tiering / Glacier tiers.
  • EBS: gp3 desacopla perf/tamaño; io2/io2 BX para misión crítica; st1/sc1 para throughput/frío.
  • EFS: regional, NFS, multi-AZ; IA/Archive tiers; miles de montajes concurrentes.
  • DynamoDB: PK + sort key; GSIs/LSIs; DAX; on-demand/provisioned; streams para eventos.
  • RDS/Aurora: Multi-AZ para HA; read replicas; backups; Proxy para pool de conexiones.

Security

  • IAM: users/roles/policies; least privilege; MFA; SCPs en Organizations.
  • KMS: CMKs, envelope encryption, rotación automática, audit con CloudTrail.
  • Secrets Manager: rotación (motores RDS), IAM fino + endpoint policies.
  • Security Hub & GuardDuty: findings centralizados y detección de amenazas.

Observability

  • CloudWatch: métricas, logs, alarmas, dashboards, Logs Insights.
  • CloudTrail: auditoría (management/data events, Insights).
  • Config: inventario, drift, cumplimiento con reglas/aggregators.

Cost Optimization

  • Right-sizing con Compute Optimizer; Savings Plans/RIs para steady-state; Spot para stateless.
  • Reduce NAT: S3/DynamoDB con Gateway Endpoints.
  • S3 Intelligent-Tiering + lifecycle a Glacier; retención de CloudWatch Logs + export a S3.

Exam Tips

  • Lee primero requisitos (HA, RTO/RPO, rendimiento, cumplimiento, coste) y descarta lo que los rompe.
  • Prefiere multi-AZ/gestionado por defecto; multi-Region solo cuando lo pidan (latencia/DR).
  • Para “sin Internet público”: piensa VPC Endpoints antes que NAT/IGW.
  • Cachea antes de escalar BD; desacopla con SQS/Kinesis para picos.
