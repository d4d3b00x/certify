AWS SAA-C03 — Domain 3 (Design High-Performing Architectures)

Low latency, high throughput — architect for speed

Domain 3 is about choosing services and patterns that deliver snappy latency, big throughput, and smart scaling. Not “bigger server” — better architecture.

Patterns that score

  • Scale-out with ASG/containers/serverless instead of vertical scaling.
  • Cache everywhere: CloudFront, API caching, ElastiCache (Redis/Memcached).
  • Right storage: EBS gp3/io2, EFS for shared POSIX, S3 for massive throughput.
  • Parallelize using SQS, Kinesis, Step Functions, and Lambda concurrency.
  • Accelerate network with Global Accelerator, placement groups, and ENA.
Tip: “Bursty workload” → serverless or ASG + queue/stream to absorb spikes.

Common pitfalls

  • Vertical scaling instead of horizontal.
  • Under-provisioned EBS IOPS/throughput for your database tier.
  • No CDN for global users; serving everything from a single Region.
  • Chatty synchronous microservices lacking caching/batching.
  • Forcing relational DBs on key-value or ultra-write use cases (consider DynamoDB/Kinesis).

Study checklist

  • EC2 families & Graviton; Auto Scaling policies.
  • EBS types; EFS vs FSx; S3 performance (multi-part, prefixes).
  • RDS vs Aurora vs DynamoDB; read replicas; partitions; WCU/RCU.
  • CloudFront behaviors; Global Accelerator vs Route 53.
  • SQS/SNS/Kinesis/MSK; Lambda concurrency; DLQs.

Exam Focus

  • Prefer managed/serverless for performance with less ops.
  • Edge + caching for reads; batch/stream for heavy writes.
  • Match data model to access patterns (don’t over-SQL everything).
  • Balance performance and cost (Graviton, gp3, S3 Intelligent-Tiering).

Key AWS services

  • EC2 + ASG, ECS/EKS, Lambda
  • ElastiCache, CloudFront, Global Accelerator
  • S3, EBS, EFS, FSx
  • RDS/Aurora, DynamoDB, OpenSearch
  • SQS, SNS, Kinesis, Step Functions, EMR

Hands-on ideas

  • Benchmark gp3 vs io2 EBS on your workload.
  • Aurora + read replicas + ElastiCache for a read-heavy service.
  • Put CloudFront in front of an S3 static site and compare TTFB.
  • S3 → SQS → Lambda pipeline for parallel processing.
