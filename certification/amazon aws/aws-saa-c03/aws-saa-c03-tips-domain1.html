Design secure, simple, and private by default

Domain 1 is ~30% of the exam. It tests secure access, network isolation, encryption, and centralized visibility. Think ‚Äúroles over keys‚Äù, ‚Äúprivate over public‚Äù, and ‚Äúdefense-in-depth‚Äù.

Patterns that score

  • Short-lived creds: EC2 instance profiles, ECS task roles, IRSA for EKS.
  • Private access: prefer VPC Endpoints; use aws:SourceVpce conditions.
  • Encryption by default: KMS CMKs with least-privilege policies and grants.
  • Central visibility: Organization CloudTrail, Security Hub, GuardDuty.
Tip: If two designs work, pick the one that removes credentials or keeps traffic private.

Common pitfalls

  • Opening S3/DB to 0.0.0.0/0 instead of using endpoints.
  • Using VPC CIDR in bucket policy instead of aws:SourceVpce.
  • Mixing up SGs (stateful) vs NACLs (stateless) and stacking them oddly.
  • Hard-coding secrets vs Secrets Manager + role access.

Study checklist

  • Bucket policies with conditions: aws:SourceVpc, aws:SourceVpce, aws:PrincipalArn.
  • Gateway vs Interface endpoints; WAF vs Shield; Network Firewall vs SGs.
  • KMS key policy vs grants; rotation; CloudTrail events for KMS.

Exam Focus

  • Least-privilege with IAM roles, SCPs, resource policies.
  • Encryption choices (SSE-S3 / SSE-KMS / client-side).
  • Private connectivity (VPC endpoints, PrivateLink).
  • Network isolation (subnets, SGs vs NACLs, routes).
  • KMS + CloudTrail/CloudWatch for audit and key mgmt.

Key AWS services

  • IAM, STS, Organizations
  • KMS, Secrets Manager, ACM
  • VPC, SGs, NACLs, Network Firewall, WAF/Shield
  • S3 policies & endpoints; CloudFront OAC/OAI
  • GuardDuty, Security Hub, Macie, Config, CloudTrail

Hands-on ideas

  • S3 Gateway Endpoint + bucket policy with aws:SourceVpce.
  • IRSA on EKS (OIDC + role per service account).
  • Encrypt RDS/EBS with CMKs + rotation.
  • Enable GuardDuty + Security Hub org-wide.
