Three core tasks

  • Design secure access to AWS resources (IAM, roles, SCPs).
  • Design secure workloads and applications (VPCs, endpoints, WAF/Shield).
  • Determine appropriate data security controls (KMS, encryption, backups).

Patterns that score

  • Short-lived creds: EC2 instance profiles, ECS task roles, IRSA for EKS.
  • Private access: prefer VPC Endpoints; use aws:SourceVpce conditions.
  • Encryption by default: KMS CMKs with least-privilege policies and grants.
  • Central visibility: Org CloudTrail, Security Hub, GuardDuty.
Tip: If two designs work, choose the one that removes credentials or keeps traffic private.

Common pitfalls

  • Opening S3/DB to 0.0.0.0/0 instead of using endpoints.
  • Using VPC CIDR in bucket policy instead of aws:SourceVpce.
  • Mixing up SGs (stateful) vs NACLs (stateless).
  • Hard-coding secrets vs Secrets Manager + roles.

Study checklist

  • Bucket policies: aws:SourceVpc, aws:SourceVpce, aws:PrincipalArn.
  • Gateway vs Interface endpoints; WAF vs Shield.
  • KMS key policy vs grants; rotation; CloudTrail for KMS.

Exam Focus

  • Least-privilege with IAM roles, SCPs, resource policies.
  • Encryption choices (SSE-S3/SSE-KMS/client-side).
  • Private connectivity (VPC endpoints, PrivateLink).
  • Network isolation (subnets, SGs vs NACLs, routes).
  • KMS/CloudTrail/CloudWatch for key mgmt & audit.

Key AWS services

  • IAM, STS, Organizations
  • KMS, Secrets Manager, ACM
  • VPC, SGs, NACLs, Network Firewall, WAF/Shield
  • S3 policies & endpoints; CloudFront OAC/OAI
  • GuardDuty, Security Hub, Macie, Config, CloudTrail

Hands-on ideas

  • S3 Gateway Endpoint with aws:SourceVpce bucket policy.
  • IRSA on EKS with OIDC.
  • Encrypt RDS/EBS with CMKs + rotation.
  • Enable GuardDuty + Security Hub org-wide.
