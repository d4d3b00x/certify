What “secure architectures” mean in AWS SAA Certification

Expect scenario questions where you pick the most secure and simplest design: roles instead of long-lived keys, private connectivity instead of public endpoints, and defense-in-depth with monitoring.
The AWS Certified Solutions Architect – Associate focuses on designing cost- and performance-optimized solutions. It’s ideal for those with AWS or strong on-premises IT experience.

Patterns that score

  • Short-lived creds: EC2 instance profiles, ECS task roles, IRSA for EKS.
  • Private access: prefer VPC Endpoints to avoid NAT/data egress and to apply aws:SourceVpce or source VPC conditions in resource policies.
  • Encryption by default: use KMS CMKs with least-privilege key policies and grants.
  • Central visibility: CloudTrail Organization trail, Security Hub, GuardDuty.
Tip: When two designs are equally functional, choose the one that removes credentials (e.g., role over access keys) or keeps traffic private (endpoint over public).

Common pitfalls

  • Allowing 0.0.0.0/0 to private resources when a VPC endpoint would be safer and cheaper.
  • Opening an S3 bucket by VPC CIDR instead of using aws:SourceVpce.
  • Confusing NACLs (stateless) with Security Groups (stateful) and stacking them incorrectly.
  • Putting secrets in env vars or code instead of Secrets Manager + role access.

Study checklist

  • Bucket policies with conditions: aws:SourceVpc, aws:SourceVpce, aws:PrincipalArn.
  • Compare endpoint types: Gateway (S3/DDB) vs Interface (most services).
  • WAF vs Shield, Network Firewall vs Security Groups.
  • CMK key policy vs grants; rotation; CloudTrail logs for KMS.

Exam Focus

  • Design least-privilege access using IAM roles, SCPs, and resource policies
  • Choose encryption options (KMS CMKs, SSE-S3/SSE-KMS, client-side)
  • Private connectivity (VPC endpoints: Gateway/Interface, PrivateLink)
  • Network isolation (subnets, NACLs vs Security Groups, route tables)
  • Key management, rotation, and audit with CloudTrail/CloudWatch

Key AWS services

  • IAM, STS, Organizations (SCPs, delegated admin)
  • KMS, Secrets Manager, AWS Certificate Manager (ACM)
  • VPC, NACLs, Security Groups, Route 53, Network Firewall, WAF + Shield
  • S3 bucket & VPC endpoint policies; CloudFront OAC/OAI
  • GuardDuty, Security Hub, Macie, Config, CloudTrail

Hands-on ideas

  • Create an S3 Gateway VPC Endpoint with a bucket policy using aws:SourceVpce
  • IRSA on EKS: map a service account to an IAM role with OIDC
  • Encrypt an RDS and EBS stack with KMS CMKs + automatic rotation
  • Enable GuardDuty + Security Hub across an AWS Organization and view findings
