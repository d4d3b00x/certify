Cost optimization mindset

Designs should minimize idle and egress while picking the right pricing model. Use managed/serverless where possible and observe costs continuously.

Patterns that score

  • Savings Plans for steady compute; mix with Spot for bursty/interruptible work.
  • Private endpoints for S3/DynamoDB to eliminate NAT data processing fees.
  • Lifecycle & retention to push cold logs/objects to Glacier classes.
  • Compute Optimizer and right‑sizing across EC2/EBS/Lambda/ECS.
Tip: Always ask “can I avoid running this?” (event‑driven/serverless) before “which instance type?”.

Pitfalls

  • Keeping large NAT traffic for S3/DynamoDB rather than using Gateway endpoints.
  • Infinite CloudWatch Logs retention; no lifecycle/export to S3.
  • Over‑provisioned gp2 volumes when gp3 with tuned IOPS/throughput is cheaper.

Study checklist

  • Savings Plans vs RIs; coverage vs utilization metrics.
  • S3 storage classes and minimum storage durations / retrieval fees.
  • Cross‑AZ/Region data transfer patterns and how to minimize them.

Exam Focus

  • Choose the right pricing model (Savings Plans, RIs, Spot)
  • Use serverless/managed services to reduce undifferentiated load
  • Control data transfer and NAT costs with private endpoints
  • Lifecycle, retention, and right‑sizing across compute and storage

Key AWS services

  • Savings Plans, Reserved Instances, Spot; Cost Explorer & Budgets
  • Compute Optimizer, Trusted Advisor, SSM Automation
  • S3 Intelligent‑Tiering + lifecycle to Glacier classes
  • VPC Gateway/Interface Endpoints (S3/DynamoDB to cut NAT)
  • CloudWatch Logs retention, Logs Insights, Export to S3

Hands‑on ideas

  • Create a 3‑year Compute Savings Plan and simulate coverage on sample usage
  • Add Gateway endpoints for S3/DynamoDB; compare NAT data processing costs
  • Enable S3 Intelligent‑Tiering + lifecycle transitions to Glacier IA/DA
  • Run Compute Optimizer and apply right‑size recommendations
