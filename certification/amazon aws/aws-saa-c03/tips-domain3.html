Performance themes

Performance in SAA eschews “bigger boxes” in favor of better patterns: cache, batch/parallelize, stream, and pick the right data layout and storage class.

Patterns that score

  • Cache first: Redis/Memcached; CloudFront for static & dynamic origins.
  • Columnar + partitioning in S3 for analytics (Parquet/ORC + Athena/Glue).
  • Right storage: gp3 for most SSD; io2 for critical low‑latency; EFS perf modes.
  • Global networking with Global Accelerator when responses can't be cached.
Tip: Performance and cost often align—scan fewer bytes, move less data, cache more.

Pitfalls

  • Keeping JSON/CSV for analytics, causing huge scanned bytes in Athena.
  • Scaling up a single instance instead of horizontal scaling or caching.
  • Ignoring EBS‑optimized instances and network throughput limits.

Study checklist

  • ElastiCache patterns: cache‑aside vs write‑through vs read‑through.
  • Throughput vs IOPS; gp3 tunables; io2 Block Express benefits.
  • CloudFront behaviors, OAC/OAI, origin groups, TA vs CloudFront.

Exam Focus

  • Choose services and patterns to reduce latency and increase throughput
  • Use caches (ElastiCache/CloudFront) and appropriate storage classes
  • Right‑size compute, select volume types, and parallelize work
  • Design for global users (Global Accelerator, multi‑Region read local)

Key AWS services

  • ElastiCache (Redis/Memcached), CloudFront, S3 TA + multipart upload
  • Athena/Glue, EMR, Redshift; Parquet/ORC; partitioning
  • EC2 instance families, EBS gp3/io2, EFS performance modes
  • Aurora (Global Database, reader endpoints), DynamoDB DAX
  • Global Accelerator, ALB/NLB tuning

Hands‑on ideas

  • Add Redis cache‑aside to an RDS‑backed app; measure p95 latency
  • Convert CSV to Parquet in S3 and query with Athena + Glue catalog
  • Benchmark EBS gp3 vs io2 with FIO; test EBS‑optimized instances
  • Enable S3 Transfer Acceleration + multipart upload for large files
