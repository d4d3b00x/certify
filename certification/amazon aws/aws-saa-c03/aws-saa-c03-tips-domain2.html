Build for high availability, fault tolerance, and graceful degradation. Pick designs that survive AZ failures, recover quickly, and avoid single points of failure.

Core tasks:

  • Design multi-tier architectures that survive failures (ALB + ASG + Multi-AZ).
  • Decouple with SQS/SNS/EventBridge so one tier can fail without taking down the rest.
  • Select storage/DB features for durability and recovery (S3 CRR, RDS Multi-AZ, backups/PITR).

Patterns that score

  • Stateless app tier behind an ALB with ASG across ‚â•2 AZs.
  • Stateful tiers with Multi-AZ (RDS/Aurora/EFS), and read replicas for scale.
  • Decouple with queues/events to isolate failures and smooth spikes.
  • Global reach with CloudFront, S3 CRR, and Route 53 failover/latency policies.
  • Backups + DR tested: PITR, cross-region copies, recovery runbooks.

Common pitfalls

  • Single-AZ databases or single-instance apps.
  • Sticky sessions without shared/session store (use ElastiCache or JWTs).
  • Tight synchronous coupling between services (no retries/backoff/DLQs).
  • Manual failover scripts instead of managed features (RDS/Aurora auto failover).
  • Ignoring health checks and deregistration delay on ALB/Target Groups.

Study checklist

  • ALB vs NLB vs GWLB; health checks; deregistration delay; slow start.
  • ASG policies (target tracking/step/scheduled), cooldowns, warm pools.
  • RDS/Aurora: Multi-AZ vs read replicas, backups, Global Database, failover times.
  • S3 versioning, CRR/SRR, object lock and retention.
  • Route 53 policies: failover, latency, weighted, geolocation + health checks.

Exam Focus

  • Choose multi-AZ vs multi-Region appropriately (DR requirements).
  • Graceful degradation with retries/backoff and circuit breakers.
  • Decouple services with SQS/SNS/EventBridge; use DLQs.
  • Prefer managed failover (Aurora, RDS, Global Accelerator) over DIY.

Key AWS services

  • ALB/NLB, Auto Scaling, Launch Templates
  • RDS, Aurora, DynamoDB global tables, ElastiCache
  • S3, EFS, FSx, AWS Backup
  • Route 53, CloudFront, Global Accelerator
  • SQS, SNS, EventBridge, Step Functions

Hands-on ideas

  • ALB + ASG across 2 AZs; terminate instances to see healing.
  • Enable RDS Multi-AZ and simulate failover.
  • Route 53 failover between two endpoints (active/passive).
  • SQS between producers and Lambda consumers with DLQ.
