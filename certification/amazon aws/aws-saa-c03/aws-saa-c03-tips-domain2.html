AWS SAA-C03 — Domain 2 (Design Resilient Architectures)

Build for failure: multi-AZ, decoupled, and self-healing

Domain 2 is all about high availability, fault tolerance, and graceful degradation. Expect scenarios that survive AZ loss, recover quickly, and avoid single points of failure.

Patterns that score

  • Stateless app tier behind an ALB with ASG across ≥2 AZs.
  • Stateful tiers with Multi-AZ (RDS/Aurora/EFS) and read replicas for scale.
  • Decouple with SQS/SNS/EventBridge to isolate failures and smooth spikes.
  • Global reach via CloudFront, S3 CRR, and Route 53 latency/failover policies.
  • Backups + DR tested: PITR, cross-Region copies, recovery runbooks.
Tip: “Zero downtime during AZ failure” → Multi-AZ + health checks + automatic failover.

Common pitfalls

  • Single-AZ databases or single-instance apps.
  • Sticky sessions without shared/session store (ElastiCache or stateless JWTs).
  • Tight synchronous coupling (no retries/backoff/DLQs).
  • DIY failover scripts instead of managed features (Aurora, RDS).
  • Ignoring health checks and deregistration delay on target groups.

Study checklist

  • ALB vs NLB vs GWLB; health checks; deregistration delay; slow start.
  • ASG policies (target tracking/step/scheduled), cooldowns, warm pools.
  • RDS/Aurora: Multi-AZ vs read replicas, backups, Global Database, failover times.
  • S3 versioning, CRR/SRR, object lock and retention.
  • Route 53 policies: failover, latency, weighted, geolocation + health checks.

Exam Focus

  • Choose multi-AZ vs multi-Region based on RTO/RPO.
  • Graceful degradation with retries/backoff and circuit breakers.
  • Decouple services with SQS/SNS/EventBridge; use DLQs.
  • Prefer managed failover (Aurora/RDS/Global Accelerator) over DIY.

Key AWS services

  • ALB/NLB, Auto Scaling, Launch Templates
  • RDS, Aurora, DynamoDB global tables, ElastiCache
  • S3, EFS, FSx, AWS Backup
  • Route 53, CloudFront, Global Accelerator
  • SQS, SNS, EventBridge, Step Functions

Hands-on ideas

  • ALB + ASG across 2 AZs; terminate instances to see self-healing.
  • Enable RDS Multi-AZ and simulate failover.
  • Route 53 failover between two endpoints (active/passive).
  • SQS between producers and Lambda consumers with DLQ.
