Azure AZ-104 — Domain 1 (Manage Azure identities & governance)

Secure by role, organized by scope, governed by policy

Domain 1 is ~15–20% of the exam. You’ll manage Entra ID objects, scopes (management groups, subscriptions, resource groups), RBAC, Policy/initiatives, resource locks, tags, and cost controls. Think “groups over individuals”, “least privilege”, and “guardrails with Policy”.

Patterns that score

  • Assign to groups, not users: create Azure AD (Entra ID) groups and grant role assignments at the right scope (MG/subscription/RG/resource).
  • Use built-in roles first: Reader, Contributor, Owner, User Access Administrator; create customRole only when necessary.
  • Govern with Policy: enforce allowed locations/SKUs, require tags, deny public IPs; bundle as initiatives and assign at MG or subscription.
  • Protect critical resources: use CanNotDelete and ReadOnly resource locks for stateful protection.
  • Budget & alerts: set Cost Management budgets with email/Action Group alerts, and review Advisor recommendations.
  • Elevated access via PIM: Privileged Identity Management for just-in-time roles and access reviews (enforce MFA, approvals).
Exam hint: If two solutions work, choose the one that assigns RBAC to a group at the highest safe scope and enforces guardrails with Policy.

Common pitfalls

  • Giving Owner broadly instead of least-privilege (e.g., Contributor + specific Reader on sensitive data).
  • Putting tags on resources but not on resource groups, then failing inheritance for policies requiring tags.
  • Creating policies but forgetting to remediate drift with modify/deployIfNotExists or an assignment with remediation task.
  • Confusing directory roles (Entra ID) with Azure RBAC roles (resource access).
  • Applying locks at resource level only; prefer RG or subscription scope when appropriate.

Study checklist

  • Entra ID objects: users, groups (assigned vs dynamic), service principals, app registrations, guest users (B2B).
  • Scopes & inheritance: management groups → subscriptions → resource groups → resources.
  • RBAC: role definitions vs assignments; deny assignments; built-in roles; custom roles JSON.
  • Azure Policy: definitions vs initiatives; effects (deny, audit, append, modify, deployIfNotExists); parameters.
  • Resource organization: naming, tags, locks, blueprints (legacy) vs policy + templates.
  • Cost Management: budgets, exports, tags for chargeback, Advisor.
  • PIM, access reviews, Conditional Access basics (MFA for privileged roles).

CLI quick hits

  • Create RG: az group create -n rg-core -l westeurope
  • Assign RBAC: az role assignment create --assignee <objectId> --role Reader --scope <scopeId>
  • List role defs: az role definition list -o table
  • Policy assign: az policy assignment create --name require-tags --policy <policyId> --scope <scopeId> --params @params.json
  • Budget: az consumption budget create --amount 500 --time-grain Monthly --name app1 --category Cost --scope <subId-scope>
  • Lock RG: az lock create --name protect --lock-type CanNotDelete --resource-group rg-core

Exam Focus

  • Manage Entra ID: users, groups, device settings, external users.
  • Control access with Azure RBAC at the right scope and verify effective permissions.
  • Implement governance with Policy/initiatives, tagging strategies, and locks.
  • Organize with management groups & subscriptions; apply guardrails at MG for inheritance.
  • Monitor and control costs with budgets, exports, and Advisor.

Key Azure services

  • Microsoft Entra ID (Azure AD), PIM, Access Reviews
  • Management Groups, Subscriptions, Resource Groups
  • Azure RBAC (role definitions & assignments)
  • Azure Policy (definitions, initiatives, remediation)
  • Resource Locks & Tags, Naming standards
  • Cost Management + Billing, Advisor

Hands-on ideas

  • Create an MG hierarchy (mg-root → mg-prod/mg-dev), then assign a tagging initiative at mg-prod.
  • Build a custom role allowing read on Compute + start/stop VMs only; assign at RG scope to a group.
  • Require tags via Policy (environment, owner) and remediate existing resources.
  • Apply CanNotDelete lock to a shared RG and test deletion behavior.
  • Create a monthly budget with alert at 80% and 100%; filter by tag for a specific app.
