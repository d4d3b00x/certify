Domain 2 — Implement and manage storage (15–20%)

Storage accounts · Security · Redundancy · Lifecycle · Files/Blob

Storage choices and security

Pick the right account kind and redundancy, then enforce private access, encryption, and lifecycle to reduce risk and cost.

Patterns that score

  • Private Endpoints to Blob/Files/Queues; disable public network access.
  • Redundancy: LRS for dev, ZRS for zone resiliency, GRS/GZRS for cross‑region DR; RA‑* for read access.
  • Access: RBAC for management; SAS (user/delegation) for scoped data access; POSIX ACLs for hierarchical namespace (Data Lake Gen2).
  • Protection: soft delete + versioning; blob immutability policies (legal/locked) for retention.
  • Azure File Sync: cache hot data on‑prem; cloud tiering; Kerberos/AD DS integration.
Tip: If the scenario says “prevent data exfiltration,” combine Private Endpoints with Private DNS zones and restrict egress via Azure Firewall.

Common pitfalls

  • Using account keys broadly instead of SAS tokens or RBAC.
  • Relying on service endpoints when private access is mandated.
  • Forgetting to enable soft delete for both containers and blobs; missing file share soft delete.

Study checklist

  • Account types (StorageV2, BlobStorage, BlockBlobStorage, FileStorage); performance (Standard/Premium); tiers (Hot/Cool/Archive).
  • Data protection: versioning, soft delete, change feed.
  • Networking: PE vs Service Endpoints; Private DNS zones; deny public network access.

Exam Focus

  • Create storage accounts with proper redundancy and private connectivity
  • Generate SAS, configure RBAC, and set ACLs for Data Lake
  • Enable lifecycle management and immutability policies
  • Configure Azure Files with AD authentication and Azure File Sync

Key Azure services

  • Storage Accounts (Blob, Files, Queues, Tables)
  • Private Endpoints, Private DNS Zones, Service Endpoints
  • Defender for Storage, Encryption with CMK/PMK
  • Azure Migrate/Copy for data movement, AzCopy, Storage Explorer

Hands‑on ideas

  • Create a GZRS account with Private Endpoint and deny public network access
  • Enable versioning + soft delete + immutability on a container
  • Configure Azure File Sync with cloud tiering and AD‑based SMB
  • Author lifecycle rules to archive older versions after 30 days
