Domain 4 — Configure and manage virtual networking (25–30%)

VNets · Subnets · NSGs/ASGs · Routing · Private Endpoints · LB/AppGW · DNS

Networking design

Build hub‑and‑spoke topologies with central firewalls, private endpoints for PaaS, and explicit routing. Prefer deny by default with NSGs.

Patterns that score

  • Segmentation: NSGs per subnet + ASGs per workload; inbound least privilege.
  • Private access: Private Endpoints + Private DNS; disable public endpoints on Storage/SQL.
  • Routing: UDRs to send traffic through NVA/Azure Firewall; 0.0.0.0/0 to firewall for egress filtering.
  • Connectivity: VNet peering (non‑transitive); use hub VNet with Firewall and Virtual WAN for scale.
  • Load balancing: LB (L4) for VMs/AKS; App Gateway (L7,WAF,SNI); Front Door for global anycast + TLS offload.
Tip: If compliance says “no data over public internet,” combine Private Endpoints, DNS resolution, and UDR egress through Firewall/ER.

Common pitfalls

  • Relying on service endpoints when exfiltration risk must be mitigated.
  • Mismatched DNS zones for private endpoints causing intermittent failures.
  • Forgetting health probes/NSG allowances for Load Balancers and App Gateway.

Study checklist

  • NSGs vs Azure Firewall; ASGs; DDoS Protection plans; Route Tables & BGP concepts.
  • Private DNS zones and links; name resolution order.
  • ExpressRoute vs VPN; ER circuits (SKUs, peering types, route filters).

Exam Focus

  • Create and secure VNets/subnets, NSGs/ASGs, and route tables
  • Configure Private Endpoints + Private DNS; disable public access
  • Implement load balancing (LB/App Gateway) and application routing
  • Design hybrid connectivity (VPN/ExpressRoute) with hub‑and‑spoke

Key Azure services

  • VNet, Subnets, NSG/ASG, Route Tables, Azure Firewall, DDoS Protection
  • Private Endpoints, Private DNS, Service Endpoints
  • Load Balancer, Application Gateway (WAF), Front Door, Traffic Manager
  • VPN Gateway, Virtual WAN, ExpressRoute

Hands‑on ideas

  • Build hub‑and‑spoke with Azure Firewall and route all egress via the hub
  • Create Private Endpoints for Storage/SQL; validate DNS resolution
  • Deploy App Gateway WAF v2 with end‑to‑end TLS to a VMSS backend
