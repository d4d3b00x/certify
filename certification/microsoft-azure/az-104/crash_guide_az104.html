Microsoft Certified: Azure Administrator Associate (AZ-104)

Crash Guide

One‑page recap of Azure services, patterns, and exam‑day heuristics.

Focus on least privilege, automation, and resilience across identity, storage, compute, networking, and monitoring.

Exam Functional Areas

  • 1 — Manage Azure identities and governance (25–30%): Entra ID (Azure AD), RBAC, PIM, Privileged Identity Management, policy & blueprint alternatives (Azure Policy, initiatives), subscriptions & management groups.
  • 2 — Implement and manage storage (15–20%): Storage accounts (Blob/File/Queue/Table), access (SAS, RBAC, ACLs), redundancy (LRS/ZRS/GRS/GZRS), lifecycle & security (Soft Delete, versioning, immutability).
  • 3 — Deploy and manage Azure compute resources (20–25%): VMs (images, sizes, disks), VMSS, Availability Sets/Zones, containers (ACI/AKS basics), updates & extensions, Azure App Service.
  • 4 — Configure and manage virtual networking (25–30%): VNets, subnets, NSGs/ASGs, routing & peering, VPN/ExpressRoute, Private Endpoints, Load Balancers & Application Gateways, DNS.
  • 5 — Monitor and maintain Azure resources (10–15%): Azure Monitor, Log Analytics, alerts, Update Manager, Backup, Site Recovery, cost management and automation.

Know the defaults, limits, and where controls apply (tenant/subscription/resource group/resource). Understand identity vs data‑plane permissions.

Quick Patterns

  • Private access: Private Endpoints to storage/databases; disable public access when possible.
  • Least privilege: Use built‑in RBAC roles or custom with minimal actions; enable PIM for privileged roles.
  • Resilient storage: Choose GZRS/RA‑GZRS for critical data; enable soft delete + versioning.
  • Compute HA: Use Availability Zones or Sets; VMSS with autoscale; regional pairs for DR.
  • Cost control: Budgets + alerts; Advisor recommendations; start/stop schedules for non‑prod.

Identity & Governance

  • Entra ID: users, groups (assigned/dynamic), enterprise apps (service principals), app registrations; conditional access.
  • RBAC: role assignments = who (principal) + what (role) + where (scope).
  • PIM: just‑in‑time elevation; approval, MFA, and access reviews; audit assignments.
  • Policy: enforce tags, location, SKU, allowed resource types; initiatives for bundles; Deny, Audit, DeployIfNotExists.

Networking

  • NSGs (stateful) + ASGs; UDRs for custom routing; Service Endpoints vs Private Endpoints (prefer Private for data exfil prevention).
  • Load Balancer (L4), Application Gateway (L7 + WAF), Front Door (global entry), Traffic Manager (DNS routing).
  • Hybrid: VPN (S2S/P2S) vs ExpressRoute; BGP and route filters; hub‑and‑spoke with Azure Firewall.

Storage & Data

  • Blob: tiers (Hot/Cool/Archive), immutability (legal/locked), lifecycle rules, static website, blob versioning & soft delete.
  • Files: SMB/NFS, AD DS/Entra Kerberos auth, Azure File Sync cloud tiering, private endpoints.
  • Security: Defender for Storage, encryption at rest (Microsoft‑managed vs CMK), TLS, Shared Key vs SAS vs RBAC.

Observability & Operations

  • Azure Monitor: Metrics, Logs (Kusto/KQL), Alerts, Action Groups, Workbooks, Application Insights.
  • Update Manager: maintenance configurations, schedules, pre/post scripts, assessment vs deployment.
  • Backup: vaults, policies, soft delete, immutable vaults, restore points; ASR for VM replication and failover.

Cost Optimization

  • Reservations (1/3‑year), Savings Plans, Spot VMs for interruptible workloads.
  • Storage lifecycle & archive tiers; cross‑region transfer awareness; Advisor recommendations.
  • Log data plans & table‑level retention to control ingestion costs.

Exam Tips

  • Prefer managed + policy‑enforced solutions. If two designs fit, pick the one with private access, least privilege, and automation.
  • For “no public internet,” think Private Endpoint and deny public network access at the resource level.
  • Use availability zones for HA; regional pairs for DR; VMSS autoscale over manual scale.
