Domain 1 — Manage Azure identities and governance (25–30%)

Entra ID · RBAC · PIM · Policy · Subscriptions & Management Groups

What “identities and governance” mean in AZ‑104

Expect scenario questions where you must minimize privileges, enforce controls at the correct scope (management group → subscription → resource group → resource), and enable just‑in‑time access for admins.

Patterns that score

  • RBAC at the lowest scope: prefer built‑in roles; create custom roles only when necessary with explicit Actions/NotActions.
  • PIM for privileged roles: eligible assignments, approval/MFA, time‑bound activation, access reviews, audit logs.
  • Conditional Access: require MFA for privileged roles; block legacy protocols; location/device controls.
  • Azure Policy: enforce tags, allowed locations/SKUs, require private endpoints, apply diagnostic settings with DeployIfNotExists.
  • Resource organization: management groups by environment/region; subscriptions for isolation/quotas/billing; RGs by lifecycle.
Tip: Always include the scope in your answer. Many wrong answers put a correct role on the wrong scope.

Common pitfalls

  • Using account‑level owner instead of least‑privileged built‑in roles.
  • Granting data‑plane permissions via RBAC when the service requires separate ACLs (e.g., Blob POSIX ACLs, SQL logins).
  • Leaving break‑glass accounts without conditional access exclusions or without long, monitored secrets.

Study checklist

  • Role assignment components; built‑in roles: Owner, Contributor, Reader, User Access Administrator.
  • Groups: assigned vs dynamic; service principals vs managed identities; app registrations vs enterprise apps.
  • Azure Policy effects: Deny, Audit, Append, Modify, DeployIfNotExists.
  • Blueprint alternatives: use Policy + ARM/Bicep templates; landing zone concepts.

Exam Focus

  • Assign RBAC roles properly; troubleshoot access via Check Access
  • Onboard PIM; configure approvals, MFA, justification, and time limits
  • Create and remediate Policy assignments & initiatives
  • Design subscription hierarchy with management groups

Key Azure services

  • Microsoft Entra ID (Azure AD), Conditional Access, Identity Protection
  • RBAC, PIM, Access Reviews
  • Azure Policy, Initiatives, Resource Graph, Tags
  • Management Groups, Subscriptions, Resource Groups

Hands‑on ideas

  • Create a custom role with minimal Actions; assign at RG scope and test.
  • Enable PIM for Subscription Owner; enforce approval + MFA + time limit.
  • Deploy a policy initiative that denies public network access and deploys diagnostics.
  • Organize subscriptions under management groups (Corp/Prod/Dev) with distinct policies.
